## 2026-03-26 前后端联调问题复盘

### 问题描述
测试完整版时，前端页面无法显示数据库中的35条梦境记录。

### 根本原因分析

#### 1. CORS 跨域问题（主要耗时点）
- **现象**: 浏览器控制台显示 CORS 错误，前端无法访问后端 API
- **原因**: 前端运行在 localhost:3000，后端运行在 localhost:8000，浏览器阻止跨域请求
- **尝试的失败方案**:
  - 在 FastAPI 中配置 CORSMiddleware（无效）
  - 使用 wildcard `*` 作为 allow_origins（无效）
  - 添加自定义 @app.middleware("http")（无效）
  - 在路由中手动添加 CORS headers（无效）

#### 2. Next.js 代理响应格式不匹配
- **现象**: API 返回数据但前端显示为空
- **原因**:
  - 后端返回: `{"total":35,"dreams":[...]}`
  - Next.js proxy 代理后: `[...]` (直接返回数组)
  - 前端代码: `data.dreams` (访问 undefined，返回空数组)
- **解决**: 修改 `lib/api-client.ts` 的 `fetchDreams` 函数，同时处理两种格式

#### 3. 环境变量配置
- 需确保 `NEXT_PUBLIC_DEMO_MODE=false` 关闭演示模式

### 最终解决方案

```typescript
// next.config.ts - 使用 Next.js rewrites 代理 API 请求
async rewrites() {
  return [
    {
      source: '/api/:path*',
      destination: 'http://localhost:8000/api/:path*',
    },
  ]
}
```

```typescript
// lib/api-client.ts - 处理两种响应格式
export async function fetchDreams(userId: number = 1): Promise<Dream[]> {
  const data = await response.json()

  // 处理两种响应格式
  if (Array.isArray(data)) {
    return data  // Next.js 代理返回数组
  } else if (data?.dreams) {
    return data.dreams  // 直接后端调用返回对象
  }
  return []
}
```

### 经验教训

1. **CORS 问题优先考虑服务器端代理**: Next.js rewrites / Nginx 反向代理 比 CORS 配置更可靠
2. **API 响应格式要统一**: 确保代理不会改变响应结构
3. **环境变量检查清单**: 测试前先确认 DEMO_MODE 等开关状态
4. **快速验证方法**: 用 curl 先测试 API 是否正常，排除后端问题
